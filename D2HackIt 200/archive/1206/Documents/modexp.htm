Introduction

D2Hackit supports "Item mod expression" since build 1116, this technique allows the user to specify item property filters in an expression, that is, a character string containing constants, mod-variables, logical operators, arithmetical operators, indices, and parenthesizes.

Operator Precedence

Operators in the same group have equal precedence and are evaluated left to right in an expression unless explicitly forced by a pair of parentheses, ( ). Operator precedence are in the following order (from highest to lowest precedence):

Operator Name or Meaning
[] indexing
+ arithmetic positive (unary)
- arithmetic negative (unary)
! logical NOT (unary)
*, / multiplication, division
+, - addition, subtraction
>, <, >=, <=, ==, != comparison
&&, || logical AND, logical OR

Mod-Variables

Each item mod (property) contains a mod ID and 4 integer values:

DWORD ModID;
int Value[4];

Mod ID distinguishes a mod from others, for example, "+x to strength" has mod ID of 0, "+x to energy" has mod ID of 2, "+x to life" has mod ID of 7, etc.

The 4 integer values contain mod values for the particular mod, and are in English reading order from left to right, below are a few examples:

Mod Description Mod ID Value[0] Value[1] Value[2] Value[3]
+20 to Strength 0 20 - - -
10% Faster Cast Rate 105 10 - - -
+3 to Meteor (Sorceress Only) 107 3 56 - -
+1 to Necromancer Skill Levels 83 1 2 - -
Level 2 Life Tap (10/15 Charges) 204 2 82 10 15

Mod Alias

It could be difficult for human being to memorize all mod ID's as raw abstract numbers, to solve this problem, we recommend you to define and use mod alias, which are intuitive character strings associated to mod ID's. Most of the useful mod's have already been defined in "D2Items.ini", please take a look at the [mod] section to have an impression on how mod alias are defined. If you need to define your own mod alias, please do it in this format:

alias_name=mod_ID[,[Value[0]],[Value[1]],[Value[2]],[Value[3]]]

Only the "alias_name" and "mod_ID" are required, all other values are optional. For example:

life_leech=60

The mod ID must be 60(x% Life Stolen per Hit), leech percentage does not matter at this point.

meteor=107,?,56

The mod ID must be 107(+x to y skill (... Only)) and Value[1] must be 56(skill ID for Meteor), skill level does not matter at this point.

lvl2_lifetap=204,2,82,?,?

The mod ID must be 204(Level x y skill (m/n Charges)), Value[0] must be 2(skill level), Value[1] must be 82(skill ID for Life Tap), remaining charges and maximum charges do not matter at this point.

In a mod alias definition, mod ID and all numeric portions must match the real item property exactly, portions those shall be ignored are denoted as "?". Portions shall be separated by commas.

Mod alias can only contain letters, digits and under-scores, and must starts with a letter or under-score. The following mod alias are valid:

my_mod1
_my_Mod1
_1_MY_mod

Whereas the following alias are invalid:

1_my_mod   // This is INVALID! Must start with a letter or under-score!

Mod alias are case-insensitive so there is no difference among "str", "STR", and "sTr".

There are a set of keywords are reserved as "built-in" variables, they have predefined meanings and must not be used for defining your own mod alias, see next paragraph for a complete list of built-in variables.

Special Mods (Built-in Variables)

Special mods are mod alias that have been predefined internally, you cannot use them to define your own mod alias, nor can you by any means modify the ways how they are defined. You may use them just as using normal alias except for that special mods do not support indexing operators ("[", "]").

Using Mod-Variables in an Expression

A direct mod-variable is used in an expression in below format:

$variable_name[value_index]

The leading "$" symbol is required in order to distinguish a mod-variable from a constant number, otherwise D2Hackit will be confused, for example, there is no way to tell if by "0" the user means the mod ID of "+x to Strength" or a plain "zero" number.

"[Value_index]" is optional, if no index is specified or the specified index is invalid, the index 0 will be used by default, so "$60" gives exactly same result as "$60[0]".

To express "sum of +strength and +dexterity must be greater than 15", you may do it as below:

$0[0] + $2[0] > 15

since both indices are [0], or you can omit them and simply write:

$0 + $2 > 15

Using Mod Alias in an Expression

A mod alias is used in an expression in below format:

alias_name[value_index]

"[Value_index]" is optional, if no index is specified or the specified index is invalid, the index 0 will be used by default, so "life_leech" gives exactly same result as "life_leech[0]".

To express "sum of +strength and +dexterity must be greater than 15", you may do it as below:

str[0] + dex[0] > 15

since both indices are [0], or you can omit them and simply write:

str + dex > 15

More Examples on Mod Expression

　

Sample 1: 3 or 4 sockets ethereal normal/superior elite body armors.

Item Codes: uui,uea,ula,utu,ung,ucl,uhn,urs,upl,ult,uld,uth,uul,uar,utp

Expression: (sockets==3 || sockets==4) && ethereal && (quality==2 || quality==3)

Remarks: Notice the Boolean style usage of mod "ethereal". If you want a non-ethereal item, use "!ethereal". You can, of course, use original "ethereal != 0" or "ethereal == 0" instead, but this isn't likely a professional C++ programmer would do, is it?

　

Sample 2: Perfect Raven Frost ring.

Item Codes: rin

Expression: quality==7 && dex==20 && ar=250

　

Sample 3: +2 any class skills, 30% magic find amulets or circlets.

Item Codes: amu,ci0,ci1,ci2,ci3

Expression: $83==2 && mf>=30

Remarks: Notice the combined usage of mod-variables and mod alias.

　

Sample 4: Level 2 Life Tap wands with 20+ maximum charges.

Item Codes: wnd,ywn,bwn,gwn,9wn,9yw,9bw,9gw,,7wn,7yw,7bw,7gw

Expression: $204==2 && $204[1]==82 && $204[3]>=20

Remarks: An expression with no mod alias used.

　

Sample 5: +2 sor/pal/nec skills, 20% faster cast rate, 15% all resistance or +30 total to str/dex or +50 total to life/mana circlets.

Item Codes: ci0,ci1,ci2,ci3

Expression: (sor==2 || pal==2 || nec==2) && fcr==20 && (all_res>=15 || str+dex>=30 || life+mana>=50)

Remarks: Notice the creative usage of the addition operator "+", we don't care how much str and dex it exactly adds, just the sum value that matters.

　

Sample 6: +2 javelin and spear skills, 20% ias, 3% life or mana leech, +40% total to fire/lightning resistance or +30 to life or +20 total to str/dex gloves.

Item Codes: lgl,vgl,mgl,tgl,hgl,xlg,xvg,xmg,xtg,xhg,ulg,uvg,umg,utg,uhg

Expression: ama_jav==2 && ias==20 && (life_leech+mana_leech>=3) && (fire_res+lit_res>=40 || life>=30 || str+dex>=20)
